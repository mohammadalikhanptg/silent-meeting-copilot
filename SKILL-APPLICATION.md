## Key design choices informed by skills

### frontend-design

- **LiveFocusCard hero hierarchy**: 3px gradient stripe on card top as a signal-accent — gives the card clear visual dominance over the coaching sub-panel without relying on size alone. Spec directive: "signal accent: indigo-to-cyan gradient for the primary live/coaching emphasis."
- **Typography elevation**: `var(--font-display)` (Bricolage Grotesque) for the `lfc-eyebrow` and `lfc-suggestion` elements. Body panels use `var(--font-sans)` (Inter). This matches Phase 1 shell headings and creates a consistent display register.
- **Drift alert animation**: `lfc-alert-pulse` uses a gentle box-shadow expansion (not flashing text) to honour the "calm overrides" pass from the Phase 2a design pass. Disabled under `prefers-reduced-motion`.
- **AlignmentMeter fill transition**: 0.8s `cubic-bezier(0.16,1,0.3,1)` ease for the fill bar — the meter updates calmly, matching the existing `balanceFill` transition in the coaching sub-panel.
- **Ready state opacity**: `smc-live-focus-card--ready` at `opacity: 0.72` rather than hiding the card — the card is always visible in the cockpit, just quieter before coaching data arrives. Never blocks the transcript.
